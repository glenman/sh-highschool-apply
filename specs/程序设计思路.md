# 上海中考志愿推荐程序：完整开发思路 + 资料准备 + 实施步骤

&#x20;

结合**上海中考名额分配（到区 / 到校）+ 统一录取**的核心政策，我给你整理一套**零基础可落地、精准贴合政策、可直接开发**的完整方案。程序的核心逻辑是：**用户输入区 + 一模成绩 → 换算位次定位 → 按政策过滤志愿资格 → 冲稳保梯度匹配 → 输出合规志愿表**，完全匹配你的需求。

***

## 一、程序核心设计思路（5 大模块，政策为根，数据为王）

整个程序的灵魂是**严格遵守上海中考招生政策**，所有推荐结果不能违背录取规则；核心是**一模成绩→位次换算**（因为各区一模难度 / 总分不同，分数无意义，**位次才是精准定位的关键**）。

### 模块 1：用户信息采集（必填 + 选填）

只收集**影响志愿资格和定位**的核心信息，不冗余：

1. **必填项**
   - 所在行政区（上海 16 区，政策 / 名额全区分开）
   - 初中性质（公办 / 民办：**民办无分配到校资格**，这是政策硬规则）
   - 一模总分（或各科分数）
   - 一模区排名（可选，没有则用分数自动换算）
2. **选填项（优化推荐）**
   - 初中学校名称（精准匹配「分配到校」名额）
   - 志愿偏好（冲刺 / 稳妥 / 保底倾向、离家远近、学校层级）

### 模块 2：成绩标准化 + 位次换算（程序核心！）

解决核心问题：**不同区一模总分不同、难度不同，分数无法直接对比**

- 输入：区 + 一模分数
- 处理：调用「各区一模分数→区内位次换算表」→ 输出**区内百分位 / 精准位次**（如区前 5%/10%/20%）
- 附加：若有校内排名，直接换算**校内位次**（分配到校只看校内排名）

### 模块 3：志愿池合规过滤（政策硬门槛，剔除无效志愿）

根据政策自动筛掉**无资格、不投放**的学校，这是「准确贴合政策」的关键：

1. 分配到校：仅保留**用户公办初中对应的到校名额学校**（民办直接清空该校志愿池）
2. 分配到区：仅保留**在用户所在区投放名额的高中**
3. 统一录取：保留所有在本区招生的普高 / 市重点 / 民办高中

### 模块 4：冲稳保梯度匹配算法（志愿推荐核心）

基于**近 3 年各校录取位次 / 分数**，按中考志愿填报的黄金规则匹配：

- **冲刺志愿**：历年录取位次比用户高 5%\~10%（有捡漏机会）
- **稳妥志愿**：历年录取位次与用户持平 ±3%（大概率录取）
- **保底志愿**：历年录取位次比用户低 10%\~15%（绝对稳妥）
- 严格遵守政策：按各区规定的**各批次志愿数量**生成（如分配到区填 3 所、到校填 2 所）

### 模块 5：结果输出（可视化志愿表）

直接生成**分批次、带备注**的志愿清单：

1. 分配到区志愿（冲 + 稳）
2. 分配到校志愿（稳 + 保，校内竞争优先稳妥）
3. 统一录取平行志愿（冲 + 稳 + 保全梯度）
4. 附加说明：推荐理由、历年录取数据、政策提示

## 二、必须预先准备的资料（分 4 类，你可逐步收集）

你能提供**政策文件 + 学校名单**是核心基础，我补充所有必需的配套数据，**全部整理成 Excel 表格即可**，无需复杂工具：

### 第一类：政策核心文件（你提供，程序的规则手册）

1. 上海市 / 各区**当年中考招生政策原文**
   - 重点：名额分配（到区 / 到校）实施细则、录取批次顺序
2. **志愿填报硬规则**
   - 各批次志愿数量（分配到区填几所？到校填几所？统一志愿最多填多少？）
   - 资格规则（公办 / 民办初中的资格差异、跨区限制）
3. **录取排序规则**
   - 区内 / 校内排名规则、同分比较规则（语文 + 数学 + 外语优先级）

### 第二类：学校基础数据（你有名单，补充标签即可）

1. 上海高中全名单（分 16 区）
2. 每所高中的**关键标签**（程序筛选用）
   - 学校层级：四校八大、市重点、区重点、优质普高、普高、民办
   - 招生批次：是否参与「分配到区 / 到校 / 统一录取」
   - 招生范围：分配到区的投放区、统一录取的招生区域
3. 公办初中名单：绑定**分配到校名额对应高中**（核心！）

### 第三类：历年录取数据（精准推荐的核心，无数据不精准）

至少准备**近 2 年**数据（越新越好）：

1. 各区分配到区：各高中→本区→最低录取分数线 / 最低位次
2. 各区分配到校：各公办初中→对应高中→录取最低分 / 位次
3. 统一录取：各高中→平行志愿→最低录取分数线 / 位次
4. 各区一模数据：
   - 一模总分值、各科分值
   - 一模分数↔区内位次换算表（如 XX 区一模 600 分 = 区前 8%）
   - 一模等效中考分换算公式

### 第四类：基础配置数据

1. 上海 16 区行政区划名单
2. 同分排序规则（政策原文提取）
3. 各批次志愿填报格式（官方模板）

## 三、具体实施步骤（从 0 到 1，分 5 阶段，新手也能做）

我按**简单→复杂**的顺序规划，你可以先做**简易版（个人用）**，再升级**进阶版（网页端）**：

### 阶段 1：吃透政策 → 翻译成程序规则（最关键！）

把你提供的政策文件，拆解成**代码 / 表格能识别的硬规则**，举例：

> 规则 1：民办初中 → 无分配到校资格，自动隐藏该批次
>
> 规则 2：分配到区录取优先级 > 分配到校 > 统一录取
>
> 规则 3：黄浦区分配到区最多填 3 所志愿，统一志愿填 15 所
>
> **所有规则整理成一张「政策规则表」，程序直接调用**。

### 阶段 2：数据结构化（把零散资料变成表格）

用 Excel 整理所有数据（后期可直接导入数据库），核心表格：

1. `学校信息表`：学校名、区、层级、招生批次、招生范围
2. `政策规则表`：批次、志愿数、资格条件、录取顺序
3. `历年录取表`：年份、区、学校、批次、最低分、最低位次
4. `一模换算表`：区、一模分数、对应区位次、等效中考分

### 阶段 3：核心算法落地（程序的大脑）

不用复杂数学，用**匹配逻辑**即可实现精准推荐：

1. **位次换算**：VLOOKUP（Excel）/ 字典匹配（Python）→ 分数转位次
2. **志愿过滤**：IF 条件判断 → 剔除无资格学校
3. **梯度排序**：按「历年录取位次」与「用户位次」的差值，分冲 / 稳 / 保

### 阶段 4：程序开发（2 种方案，按需选择）

#### 方案 A：简易版（零基础首选，1 天搞定，个人使用）

**工具**：Excel + 函数 或 Python + Streamlit（零代码网页）

- 优势：无需编程基础，数据更新方便
- 实现：用 Excel 的`VLOOKUP`/`IF`函数做匹配，直接生成志愿表；
- 进阶：用 Python 的`pandas`处理数据，`streamlit`做一个网页输入框，输入成绩自动出结果。

#### 方案 B：进阶版（网页端，多人使用）

**技术栈**（轻量，学习成本低）：

- 前端：HTML/CSS（做输入表单）
- 后端：Python（Flask 轻量框架）
- 数据库：SQLite（小型数据库，存储学校 / 录取数据）
- 核心：把阶段 3 的算法写成 Python 代码，调用数据库自动匹配。

### 阶段 5：测试 + 优化（确保精准）

1. **真实数据测试**：找已知成绩的考生，验证推荐学校是否符合历年录取
2. **政策更新**：每年中考政策微调，只需要更新「政策规则表」即可
3. **偏好优化**：加入地理位置、学校层级筛选，贴合用户需求

## 三、极简运行示例（让你直观感受效果）

**用户输入**：普陀区、公办初中、一模 600 分、无校内排名

**程序执行**：

1. 资格校验：公办→拥有「到区 + 到校 + 统一」全资格
2. 成绩换算：普陀区一模 600 分 → 区前 12%，校内预估前 10%
3. 志愿过滤：保留普陀区投放到区名额的高中、该校的到校高中
4. 梯度匹配：
   - 分配到区：冲刺（区前 8%）+ 稳妥（10%\~15%）
   - 分配到校：稳妥（校内前 10%）+ 保底
   - 统一录取：全梯度 15 所志愿
5. 输出：直接生成可填报的志愿表 + 推荐理由

### 总结

1. **核心逻辑**：政策合规为底线，一模位次换算为核心，冲稳保为推荐规则；
2. **资料关键**：政策规则 + 学校名单 + 历年录取数据 + 一模换算表（四者缺一不可）；
3. **落地方式**：先做 Excel 简易版验证逻辑，再升级 Python 网页版，成本极低、精准度拉满。
